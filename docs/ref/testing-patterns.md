# 테스트 패턴

pass@k / pass^k 패턴 기반 테스트 전략입니다.

---

## 핵심 개념

### pass@k (샘플링 기반)
> k번 시도 중 1번이라도 통과하면 성공

- **용도**: 탐색적 테스트, 경계값 발견, 비결정적 코드 검증
- **적용**: LLM 생성 코드, 랜덤 알고리즘, 확률적 로직
- **방법**: 같은 입력으로 k번 실행 → 성공률 측정

```python
# pass@k 예시: k=5 중 1회 이상 통과
results = [run_test(input) for _ in range(5)]
assert any(results), f"pass@5 실패: {results}"
```

### pass^k (반복 안정성)
> k번 연속 모두 통과해야 성공

- **용도**: 릴리즈 전 안정성 검증, 플레이키 테스트 감지
- **적용**: 비동기 코드, 타이밍 의존 로직, 네트워크 관련
- **방법**: 같은 테스트를 k번 연속 실행 → 전부 통과해야 함

```python
# pass^k 예시: k=3 연속 통과
for i in range(3):
    assert run_test(input), f"pass^3 실패 (시도 {i+1})"
```

## 작업 크기별 전략

| 작업 규모 | 전략 |
|-----------|------|
| 함수 1개 | 단위 테스트 (pass^1) |
| 모듈 | 통합 테스트 + pass^3 |
| API 엔드포인트 | E2E 테스트 + 경계값 pass@5 |
| 릴리즈 전 | pass^5 전체 스위트 |

## TDD 흐름 (`/tdd` 스킬)

```
1. 실패하는 테스트 작성 (Red)
2. 최소 구현으로 통과 (Green)
3. 리팩터링 (Refactor)
4. pass^3 확인 후 커밋
```
