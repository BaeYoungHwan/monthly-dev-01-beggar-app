# 에이전트 우선 운영 원칙

이 하네스가 Claude Code를 활용하는 방식의 철학입니다.

---

## 핵심 믿음

### 1. 에이전트는 단위 작업 전문가다
단일 에이전트가 전체 작업을 처리하는 것은 비효율적입니다.
독립적인 서브태스크는 전문 에이전트에게 위임하고, 병렬 실행합니다.

### 2. 명시적 범위가 품질을 지킨다
Claude는 요청하지 않은 기능도 구현하려 합니다.
MVP 제외 사항, `Not-done:` Trailer, 명확한 Spec으로 범위를 제어합니다.

### 3. 안전망이 먼저다
훅(hook)은 토큰을 소비하지 않습니다.
보안 차단, 감사 로그, 세션 지속성은 항상 실행되어야 합니다.

### 4. 컨텍스트는 항상 단절된다
긴 작업은 컨텍스트 압축으로 인해 중단됩니다.
`exec-plans/active/`의 `[🔄]` 상태가 재시작의 기준점입니다.

### 5. 모델 비용은 작업에 맞춰야 한다
탐색은 Haiku, 개발은 Sonnet, 설계는 Opus.
잘못된 모델 선택은 비용과 속도 모두 낭비입니다.

### 6. 검증은 증명이다
"작동하는 것 같다"는 검증이 아닙니다.
테스트, 타입 체크, 실제 실행으로만 완료를 선언합니다.

---

## 운영 원칙 요약

```
Plan → Split → Parallel → Verify → Commit → Persist
(설계) (분할) (병렬실행) (검증) (커밋) (상태저장)
```
