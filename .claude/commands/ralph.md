# /ralph — 완료 보장 루프

"완료될 때까지 멈추지 않는다"는 보장 레이어입니다.

## 언제 사용하나

- 단순히 실행하는 것이 아닌 **검증까지 완료**가 보장되어야 할 때
- 자동화된 수정-검증 루프가 필요할 때
- "한 번에 끝내라"는 작업에

## 루프 구조

```
plan → exec → verify → fix → verify → ... → success
```

각 단계:
1. **plan**: 작업 목표와 완료 기준을 명확히 정의
2. **exec**: 구현 실행
3. **verify**: 완료 기준 충족 여부 검증 (테스트, 타입체크, 실행 확인)
4. **fix**: 검증 실패 시 원인 분석 → 수정
5. **반복**: verify 통과까지 반복
6. **success**: 검증 통과 → 커밋

## 완료 기준 예시

- "모든 테스트 pass^3" (3회 연속 통과 — `docs/ref/testing-patterns.md` 참조)
- "타입 에러 0개"
- "API 응답 200 확인"
- "로컬 서버 정상 기동"

## 사용법

```
/ralph

목표: [무엇을 구현할지]
완료 기준: [언제 완료로 볼지]
```

## circuit-breaker 연동

동일 에러 3회 반복 시 자동 중단 (circuit-breaker.sh).
중단 후: 접근법 재검토 → `/ralph` 재실행.

## 주의사항

- 완료 기준 없이 시작하지 않음 — 반드시 명시
- 무한 루프 방지: 최대 시도 횟수 = circuit-breaker 설정(3회)
- 복잡도가 낮은 작업은 ralph 없이 진행 (오버킬)
