# /ultrawork — 병렬 작업 최대화

독립적인 작업을 병렬로 실행해서 토큰 효율을 20~40% 향상시킵니다.

## 언제 사용하나

- 서로 독립적인 서브태스크가 3개 이상 있을 때
- Phase 내에서 병렬 처리 가능한 작업을 최대화하고 싶을 때
- 에이전트 분산 처리로 속도를 높이고 싶을 때

## 사용법

```
/ultrawork

작업 목록:
1. [독립 작업 A]
2. [독립 작업 B]  
3. [독립 작업 C]
```

## 실행 방식

1. **의존성 분석**: 작업 간 의존관계 파악
2. **병렬 그룹 분류**:
   - Group 1: 의존성 없는 작업 (동시 실행)
   - Group 2: Group 1 완료 후 실행 가능한 작업
3. **에이전트 배분**: 각 독립 작업에 서브에이전트 할당
4. **병렬 실행**: 동일 메시지에 여러 Agent 호출
5. **결과 통합**: 각 에이전트 결과 수집 후 충돌 확인

## 병렬 처리 가능 기준

```
✅ 서로 다른 파일/모듈을 수정하는 작업
✅ 서로 다른 도메인의 API 개발
✅ 독립적인 테스트 작성
✅ 문서 생성 + 코드 구현 동시 진행

❌ 같은 파일을 수정하는 작업
❌ 이전 작업 결과가 필요한 작업
❌ 공유 상태(DB 스키마)에 동시 접근하는 작업
```

## 에이전트 모델 배분

- 탐색/분석 서브태스크 → Haiku
- 개발 서브태스크 → Sonnet
- 복잡한 설계 → Opus

## 주의사항

- 병렬 처리로 인한 파일 충돌 주의 (같은 파일 수정 금지)
- 에이전트별 완료 확인 후 통합
- 실패한 에이전트는 개별 재실행
