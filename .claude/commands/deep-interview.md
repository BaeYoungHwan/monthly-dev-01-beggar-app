# /deep-interview — 소크라테스식 아이디어 구체화

막연한 아이디어를 명확한 설계로 변환합니다.

## 목적

사용자가 "이런 걸 만들고 싶어"라고 말할 때, 구체적인 Spec이 없으면 Claude가 잘못된 방향으로 구현합니다.
이 스킬은 소크라테스식 질문으로 아이디어를 명확하게 다듬습니다.

## 진행 방식

**사용자가 아이디어를 제시하면:**

1. **이해** — 아이디어를 한 문장으로 요약해서 확인
2. **핵심 질문** — 아래 5가지 영역에서 순서대로 질문 (한 번에 1~2개)
3. **합성** — 답변을 바탕으로 PRD 초안 생성

## 5가지 질문 영역

### 1. 문제 정의
- 이것이 해결하는 핵심 문제는 무엇인가?
- 현재 어떻게 해결하고 있나? 왜 부족한가?

### 2. 사용자
- 누가 사용하나? (구체적인 페르소나)
- 주요 사용 시나리오는?

### 3. 범위 (MVP 제외 사항 발굴)
- 반드시 있어야 하는 기능은?
- **이번 버전에서 제외할 것은?** (이 질문 필수)
- "있으면 좋지만"인 기능은?

### 4. 제약
- 기술적 제약이 있나? (스택, 인프라, 예산)
- 시간 제약은?

### 5. 완료 기준
- 언제 "완성"이라고 할 수 있나?
- 성공을 어떻게 측정하나?

## 완료 시 출력

인터뷰 완료 후 자동 생성:
1. PRD 초안 (`docs/product-specs/PRD-v1.md`)
2. MVP 제외 사항 명시
3. 추천 Phase 분할안

## 주의사항

- 질문은 한 번에 1~2개만 — 사용자를 압도하지 않음
- 답변이 모호하면 구체적인 예시를 요청
- "다 중요하다"는 답변에는 우선순위 강제 (1위는 무엇인가?)
